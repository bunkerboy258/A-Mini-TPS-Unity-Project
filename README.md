项目仅作为学习交流使用
僵尸
	基于优先级驱动的状态队列有限状态机，实现全行为管控
	状态体系定义：划分 7 种核心状态
	并为每个状态配置优先级，通过状态队列动态管理待执行状态；
	外部通过Burn()、BeHit()等公共方法仅修改状态标记
	外部调用仅触发状态，业务逻辑集中在HandleXXX方法中，新增状态 / 修改行为无需改动触发逻辑
	用了一个轻量化对象池管理 对象池基于字典 + 队列” 的静态管理模式实现
tank
	坦克控制分为车体（TankControl） 和炮塔（hatchcontrol） 两大模块，前者负责移动 / 生存，后者负责瞄	准 / 武器系统，通过状态标记（isdead）联动禁用
	前进 / 后退分离限速（前进 14m/s、后退 8m/s），配合加速度 / 减速度实现非线性变速
	炮塔旋转逻辑用了两次平滑（Vector3.Lerp+Quaternion.Slerp），让旋转更丝滑
	射击逻辑是在屏幕准星处 发射一条恒长射线 用射线终点和炮口位置算出的方向向量 给予生成炮弹一个速度
	导弹不直接在炮塔脚本中实现发射逻辑，而是委托调用misslelaunercontrol的shoot方法，传递瞄准射线终点作为目标

架构
	三个主要模块
	gamemanager负责全局游戏状态、数据、流程的纯逻辑管理
	displaycontrol负责 UI、相机、交互控件的视觉表现管理
	所有操作都以状态为前置校验条件
	inputmanager负责玩家输入读取
	都是用了单例模式 方便其他模块交互
ui	
	使用了ugui实现
	每个ui界面的逻辑脚本都搭载在父容器上
	统一使用了updateuiposition()方法 在onwake（）内调用 防止分辨率改变导致错位
